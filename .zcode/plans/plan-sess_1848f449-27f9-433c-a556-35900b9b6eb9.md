## Задача
Удаление пользователя со всем прогрессом из админки. Деструктивная, необратимая операция.

## Решение

### Бэкенд — новый эндпоинт `DELETE /admin/users/{vk_id}`
**Файл:** `api/routes/admin.py` (добавить после `delete_user_dragon`, ~строка 1009)

Создаём `delete_user` — явно чистит дочерние таблицы перед удалением строки User (надёжнее, чем полагаться только на CASCADE — на случай миграции БД или отсутствия FK-constraint в уже созданной таблице):

```python
@router.delete("/users/{vk_id}")
def delete_user(vk_id: int, db: Session = Depends(get_db)):
    user = db.query(User).filter(User.vk_id == vk_id).first()
    if not user:
        raise HTTPException(status_code=404, detail="User not found")

    # Явная чистка дочерних таблиц (дублирует CASCADE для надёжности)
    from models import (UserProgress, UserLegendProgress, UserDragon,
                        UserTreasure, UserInventory, SuspiciousReport)
    db.query(UserProgress).filter(UserProgress.user_id == vk_id).delete(synchronize_session=False)
    db.query(UserLegendProgress).filter(UserLegendProgress.user_id == vk_id).delete(synchronize_session=False)
    db.query(UserTreasure).filter(UserTreasure.user_id == vk_id).delete(synchronize_session=False)
    db.query(UserInventory).filter(UserInventory.user_id == vk_id).delete(synchronize_session=False)
    db.query(SuspiciousReport).filter(SuspiciousReport.user_id == vk_id).delete(synchronize_session=False)
    db.query(UserDragon).filter(UserDragon.user_id == vk_id).delete(synchronize_session=False)

    db.delete(user)
    db.commit()
    return {"ok": True}
```

Порядок важен: `UserDragon` удаляем **последним** среди user-children, т.к. от `user_dragons.id` каскадно зависят `epic_care_state`/`epic_moodlets` — SQLite CASCADE удалит их при удалении UserDragon.

**Что НЕ трогаем:** `error_logs` (user_id не FK, nullable — исторические логи оставляем по решению), `api_request_logs` (нет колонки user_id).

### Фронтенд — двойное подтверждение + кнопка
**Файл:** `frontend/src/pages/admin/UsersList.tsx`

1. В карточке игрока (где «Копилка», «Подозрительные», «Коллекция») добавить блок «Опасная зона» с кнопкой «🗑 Удалить игрока» (класс `lair-btn-danger`, внизу карточки).
2. Двойное подтверждение:
   - **Шаг 1:** `window.confirm("Удалить игрока {first_name} {last_name} (id{vk_id})?\nВсе драконы, прогресс, баланс, сокровища будут удалены безвозвратно.")` — если Отмена, выходим.
   - **Шаг 2:** `window.prompt("Для подтверждения введи VK ID игрока ({vk_id}):")` — удаляем только если введённое значение === `String(vk_id)`. Иначе alert «ID не совпадает».
3. После успешного удаления → `client.delete('/admin/users/' + vkId)` → выход из карточки → обновление списка `users` (фильтр убранного).

Двойное подтверждение через `confirm` + `prompt` — без новой модалки, в стиле существующего UI (как `deleteDragon` использует `window.confirm`, но строже для необратимой операции).

- Не добавляю «soft delete» / архивирование — удаление физическое.
