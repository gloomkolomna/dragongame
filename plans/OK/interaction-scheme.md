```
┌──────────────────────────────────────────────────────────────────────┐
│                          ПРОЕКТ                                      │
│                                                                      │
│  ┌──────────────┐   ┌──────────────┐   ┌──────────────────────────┐ │
│  │   VK Бот     │   │  FastAPI     │   │  React SPA (Vite)        │ │
│  │  (longpoll)  │◄──┤  (uvicorn)   │──►│  /dragons/               │ │
│  │              │   │  :8001       │   │  :5173 (dev)             │ │
│  └──────┬───────┘   └──────┬───────┘   └───────────┬──────────────┘ │
│         │                  │                        │                │
│         │        ┌─────────┴─────────┐              │                │
│         └───────►│   SQLite (WAL)    │◄─────────────┘                │
│                  │   dragons.db      │                               │
│                  └───────────────────┘                               │
└──────────────────────────────────────────────────────────────────────┘
```

| Компонент | Технология | Назначение |
|-----------|-----------|------------|
| **VK Бот** | Python, vk_api, longpoll | Обработка команд игроков: активация PIN, шаги выращивания, переключение драконов |
| **FastAPI** | Python, SQLAlchemy, JWT | REST API для админки и мини-приложения |
| **React SPA** | React 18, TypeScript, Vite, VKUI | Единое приложение: админ-панель (`/admin/*`) + мини-приложение для игроков (`/`) |
| **SQLite** | WAL-режим | Хранение всех данных |

## 2. Модели данных

```
┌──────────┐       ┌───────────┐       ┌───────────────┐
│  Family  │       │  Dragon   │       │  DragonStep   │
│──────────│       │───────────│       │───────────────│
│ id       │──┐    │ id        │──┐    │ id            │
│ name     │  │    │ name      │  │    │ dragon_id FK  │
│ color    │  └───►│ family_id │  └───►│ step_number   │
│ sort_ord │       │ rarity    │       │ task_desc     │
└──────────┘       │ egg_type  │       │ magic_action  │
                   │ pin_code  │       │ hint          │
┌────────────────┐ │ egg_path  │       └───────────────┘
│ CollectionGrid │ │ dragon_path│
│────────────────│ │ steps_cnt │       ┌──────────────┐
│ id             │ │ is_active │       │    User      │
│ family_id FK   │ └───────────┘       │──────────────│
│ cell_x, cell_y │       │             │ vk_id PK     │
│ dragon_id FK ──┘       │             │ state        │
└────────────────┘       │             │ cur_dragon_id│
                          │             │ cur_step     │
┌──────────────┐         │             └──────┬───────┘
│ UserDragon   │         │                    │
│──────────────│         │         ┌──────────┴──────────┐
│ user_id FK   │─────────┘         │                     │
│ dragon_id FK ├───────────────────┘     ┌──────────────────┐
│ completed_at │                          │  UserProgress    │
└──────────────┘                          │──────────────────│
                                          │ user_id FK       │
                                          │ dragon_id FK     │
                                          │ step_number      │
                                          │ completed (bool) │
                                          │ photo_before_id  │
                                          │ photo_after_id   │
                                          └──────────────────┘
```

### Связи:
- **Dragon → Family**: N:1 (у дракона одно семейство, ON DELETE SET NULL)
- **Dragon → DragonStep**: 1:N (CASCADE)
- **CollectionGrid → Family**: N:1 (CASCADE)
- **CollectionGrid → Dragon**: N:1 (SET NULL, dragon_id UNIQUE — дракон в одной ячейке)
- **User → UserDragon**: 1:N (CASCADE)
- **User → UserProgress**: 1:N (CASCADE)
- **UserDragon**: уникальная пара (user_id, dragon_id)
- **UserProgress**: уникальная тройка (user_id, dragon_id, step_number)

## 3. Конечный автомат бота (FSM)

```
                    ┌────────────────────────────────────────┐
                    │                                        │
                    ▼                                        │
              ┌──────────┐  «Добавить дракона»  ┌──────────┐│
    запуск───►│   IDLE   │─────────────────────►│AWAIT_PIN ││
              └────┬─────┘                      └────┬─────┘│
                   │                                 │      │
     «Сменить     │                                 │ PIN  │
     дракона»     │                                 ▼      │
                   │                          ┌───────────┐ │
                   │              фото+«вышито»│GROW_STEP_1│ │
                   │                          └─────┬─────┘ │
                   │                                │       │
                   │                          фото+«вышито» │
                   │                                ▼       │
                   │                          ┌───────────┐ │
                   │                          │GROW_STEP_N│ │
                   │                          └─────┬─────┘ │
                   │                                │       │
                   │                          все шаги      │
                   │                          пройдены      │
                   │                                │       │
                   └────────────────────────────────┘       │
                                                            │
              ┌──────────────┐                              │
              │AWAIT_GARDEN  │◄──── «Сменить дракона» ──────┘
              │(выбор номера)│
              └──────┬───────┘
                     │ номер → switch_dragon → GROW_STEP_N
                     │ «не менять» / 0 → IDLE/GROW_STEP_N
                     └──────────────────────────────────────┘
```

### Переходы:

| Из состояния | Событие | В состояние |
|-------------|---------|------------|
| IDLE | Кнопка «Добавить дракона» | AWAIT_PIN |
| IDLE | Кнопка «Сменить дракона» (есть активный) | AWAIT_GARDEN |
| AWAIT_PIN | Ввод валидного 4-значного PIN | GROW_STEP_1 |
| AWAIT_PIN | Кнопка «Сменить дракона» | AWAIT_GARDEN |
| GROW_STEP_N | 2 фото + «вышито» (не последний шаг) | GROW_STEP_N+1 |
| GROW_STEP_N | 2 фото + «вышито» (последний шаг) | IDLE |
| GROW_STEP_N | Кнопка «Сменить дракона» | AWAIT_GARDEN |
| AWAIT_GARDEN | Ввод номера из списка | GROW_STEP_N (переключённый дракон) |
| AWAIT_GARDEN | «не менять» / 0 / кнопка «◀ Не менять» | IDLE или GROW_STEP_N |
| Любое | Кнопка «📖 Мой Бестиарий» | открывается мини-приложение ВК |

## 8. API endpoints — сводная таблица

### Админские (требуют JWT + VK_ALLOWED_IDS)

| Метод | Путь | Назначение |
|-------|------|-----------|
| GET | `/admin/stats` | Статистика дашборда |
| GET | `/admin/dragons` | Список всех драконов |
| GET | `/admin/dragons/{id}` | Один дракон |
| POST | `/admin/dragons` | Создать дракона (form-data, генерирует PIN) |
| PUT | `/admin/dragons/{id}` | Обновить дракона |
| DELETE | `/admin/dragons/{id}` | Удалить дракона |
| GET | `/admin/dragons/{id}/steps` | Шаги дракона |
| PUT | `/admin/dragons/{id}/steps` | Сохранить все шаги |
| GET/POST/DELETE | `/admin/families...` | CRUD семейств |
| GET | `/admin/grid?family_id=N` | Ячейки сетки |
| POST | `/admin/grid/create` | Создать сетку |
| POST | `/admin/grid/resize` | Изменить размер сетки |
| PUT | `/admin/grid/cell/{id}?dragon_id=N` | Назначить дракона в ячейку |
| DELETE | `/admin/grid/cell/{id}` | Очистить ячейку |
| GET | `/admin/users` | Список игроков |
| GET | `/admin/users/{vk_id}` | Детали игрока |
| GET | `/admin/users/{vk_id}/steps` | Шаги текущего дракона игрока |
| POST | `/admin/users/{vk_id}/steps/{n}/toggle` | Переключить шаг |
| POST | `/admin/users/{vk_id}/reset-dragon` | Сбросить прогресс |
| POST | `/admin/users/{vk_id}/dragons/{id}/restart` | Перезапустить дракона |

### Публичные (мини-приложение)

| Метод | Путь | Назначение |
|-------|------|-----------|
| GET | `/collection/{vk_id}/families` | Семейства с подсчётом собрано/всего |
| GET | `/collection/{vk_id}?family_id=N` | Сетка с состоянием ячеек |
| GET | `/dragon/{id}?vk_id=...` | Детали дракона + прогресс игрока |

### Аутентификация

| Метод | Путь | Назначение |
|-------|------|-----------|
| GET | `/auth/vk-login` | URL для VK OAuth |
| GET | `/auth/vk-callback` | Callback OAuth → JWT |
| GET | `/auth/me` | Текущий админ |
| GET | `/auth/config` | Настройки (dev login enabled?) |
| POST | `/auth/dev-login` | Локальный вход (только dev) |

## 9. Frontend — карта страниц

```
/dragons/
├── /                                    → Collection (сетка коллекции)
├── /dragon/:id                          → DragonDetail (карточка дракона)
├── /admin/login                         → Login (VK OAuth)
├── /admin                               → redirect → /admin/dashboard
├── /admin/dashboard                     → Dashboard (статистика)
├── /admin/families                      → FamiliesList (таблица семейств)
├── /admin/families/new                  → FamiliesForm
├── /admin/families/:id/edit             → FamiliesForm
├── /admin/dragons                       → DragonsList (таблица драконов)
├── /admin/dragons/new                   → DragonForm
├── /admin/dragons/:id/edit              → DragonForm
├── /admin/dragons/:id/steps             → StepsEditor
├── /admin/grid                          → GridEditor (сетка, drag-drop)
├── /admin/users                         → UsersList (игроки)
└── *                                    → redirect → /
```
