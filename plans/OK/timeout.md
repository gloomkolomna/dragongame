1. Контекст и цель
У нас есть механика выращивания драконов, состоящая из N шагов (например, 5 шагов для Ледяного дракона). На данный момент шаги выполняются последовательно без временных ограничений.

Цель: Добавить возможность указывать для каждого конкретного шага дракона временную задержку (таймаут). После завершения текущего шага пользователь НЕ может начать следующий шаг до истечения заданного времени (часы + минуты).

2. Что нужно сделать (по модулям)
А. Админ-панель (CRUD шагов)

В интерфейсе редактирования/создания шага (для каждого дракона) добавить два поля:

timeout_hours (Integer, по умолчанию 0).

timeout_minutes (Integer, по умолчанию 0).

Валидация: общее время должно быть >= 0. Если оба поля = 0 — таймаут отсутствует (старое поведение).

Отображение в списке шагов: колонка "Ожидание перед след. шагом" (например, "5ч 30м" или "Без ожидания").

Б. База данных (Миграция)

Создать миграцию для таблицы steps (или как там называется таблица шагов драконов).

Добавить столбцы:

timeout_hours (int, nullable, default 0).

timeout_minutes (int, nullable, default 0).

В. Логика бота (Серверная часть)

Модифицировать логику завершения шага.

При успешном завершении шага N (пользователь прислал фото/вышивку):

Сохраняем статус шага как "Завершен".
Смотрим настройки таймаута для этого шага.
Если таймаут > 0:
Записываем в БД (в таблицу прогресса пользователя) поле next_step_available_at = текущее время + таймаут.
Блокируем возможность отправки материалов для этого конкретного дракона до наступления этого времени.
Если таймаут = 0 — разрешаем сразу переходить к следующему шагу (как сейчас).
Г. Обработка входящих сообщений (Валидация)

Когда пользователь присылает фото и слово "вышивка" для дракона:

Проверяем, есть ли у него активный дракон с текущим шагом.
Проверяем поле next_step_available_at.
Если текущее время МЕНЬШЕ, чем next_step_available_at:
НЕ принимаем сообщение (не засчитываем шаг).
Отправляем пользователю уведомление: "❄️ Яйцо Ледяного дракона ещё созревает. Пожалуйста, подождите X ч. Y мин., прежде чем продолжить выращивание." (текст составим позже, но сейчас нужно выводить точное оставшееся время).
Если время прошло — принимаем как обычно и запускаем следующий шаг (с его собственным таймаутом).
3. Детали поведения (Важно!)
Параллельность: Таймаут применяется ТОЛЬКО к текущему дракону. Если у пользователя 3 дракона, он может спокойно выращивать второго, пока первый "отдыхает". Блокировка только на уровне user_id + dragon_id.

Отображение статуса: В команде просмотра коллекции или статуса дракона (например, /my_dragons) необходимо добавить строку:

"⏳ Следующий шаг будет доступен через: 2 часа 15 минут" (если таймаут активен).

Или "✅ Готов к следующему шагу!" (если таймаут прошел или его нет).

Команда проверки: Если пользователь пишет просто текст (не фото) для проверки статуса, бот должен отвечать с текущим оставшимся временем.

4. Логика сброса/форс-мажор
Если администратор вручную меняет статус дракона в админке (скипает шаг) — таймаут должен обнуляться и сбрасываться.

Если пользователь начинает выращивание заново (рестарт) — таймауты пересчитываются заново с первого шага.

5. Тексты сообщений (Заготовки)
При попытке отправить раньше времени:

"⏳ Этот дракон ещё отдыхает после предыдущего этапа. Осталось подождать: {hours} ч. {minutes} мин. Вернитесь позже!"

При успешном завершении шага с таймаутом:

"✅ Шаг выполнен! Следующий этап будет доступен через {hours} ч. {minutes} мин. Я уведомлю тебя, когда дракон будет готов."
